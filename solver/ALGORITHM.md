# Algorithm Overview

Reverse engineering and binary analysis often require identifying specific constant values and their usage patterns in compiled code. However, modern compilers employ aggressive optimizations that obscure direct constant usage through complex instruction sequences. For instance, division by a constant may be replaced with magic number multiplication and bit shifts, while multiplication may be decomposed into `lea` and shift operations. Traditional pattern matching approaches fail to detect these relationships because they rely on direct instruction-level analysis without understanding the semantic transformations applied to values across multiple instructions.

We present a symbolic execution-based approach that tracks register values as symbolic expressions throughout instruction execution. Our algorithm performs bidirectional symbolic execution from a target line number, maintaining a mapping of registers to their symbolic expressions (e.g., `%eax = ((%edx + %eax) >> 6) - (%eax >> 31)`). For each register, we detect relationships with the target value `V` through two mechanisms: (1) immediate value matching, where instructions directly use `V` as an operand, and (2) relationship inference, where we analyze symbolic expressions to identify arithmetic/logical relationships such as `output = input * V`, `output = input / V`, or `output = input +/- V`. To handle compiler-optimized operations like magic number divisions, we employ pattern-based detection that recognizes large constant multiplications combined with shifts and arithmetic operations, even when the relationship spans multiple registers through data movement instructions. The algorithm collects all candidate registers matching these patterns and selects the final instruction that completes the computation, ensuring we report the semantically meaningful result rather than intermediate steps.
